Atlas Suite Root Intake + Archive Rule
Official rule
Anything dropped directly into the project root is treated as an intake item, not permanent root content.

That means Atlas Suite should automatically detect, classify, process, and relocate or register those files according to project rules.

Root Intake Policy
1. Zip files dropped in root
Any .zip file placed in the root must be:

automatically detected

automatically audited

indexed in intake records

staged for extraction, review, archival, or merge workflows

preserved unless explicitly purged by policy

These are not treated as random files. They are treated as possible:

source packs

repo drops

backup payloads

vendor packages

recovered archives

migration candidates

2. Loose docs and miscellaneous files dropped in root
Any loose file such as:

.txt

.md

.json

.log

.pdf

images

spreadsheets

any non-core stray file

should be:

automatically detected

classified

added to the archive intake system

indexed for later lookup

moved or copied into archive storage according to policy

This prevents root clutter while preserving anything useful.

Archive System Rule
The archive system should become a local, gitignored preservation layer.

Purpose
It exists to:

keep the repo root clean

prevent repo bloat

preserve useful non-source artifacts

retain dropped files for audit and recovery

support project memory and historical reference

protect local-only materials that should not be committed

Archive behavior
Once fully implemented, the archive system should be:

local-first

gitignored

indexed

searchable

recoverable

version-aware where useful

Recommended root policy
Root should only contain intentional project structure
Examples:

solution files

repo folders

canonical configs

approved bootstrap files

approved scripts

Root should not become a dumping ground
Anything unrecognized or dropped there manually should go through intake processing.

Root intake rule set
Rule 1
Approved root files are left alone.

Examples:

solution files

approved readme

canonical configs

known bootstrap scripts

Rule 2
Unknown loose files in root are intake items.

Rule 3
Zip files in root always trigger archive audit workflow.

Rule 4
Loose docs and miscellaneous files are archived and indexed.

Rule 5
Archive storage becomes gitignored when the archive subsystem is active.

Rule 6
Root intake should preserve originals until policy says safe to move, copy, or delete.

Rule 7
No silent destruction of intake files.

Rule 8
All intake actions should be logged and reviewable.

Suggested intake modes
Safe mode
detect

classify

copy to archive

keep original in root

notify user

Managed mode
detect

classify

move to archive

replace with log entry or UI notification

Strict mode
block unsupported files from remaining in root

auto-route everything through archive rules

Best default for now is Safe mode.
